# ROADMAP-001 Optimaks Constitution Roadmap

Version: v1.4  
Owner: Optimaks Pte Ltd  
Status: Roadmap / Not Fully Active Yet

---

## 1. Purpose

This roadmap defines the future direction of the Optimaks Development Constitution after v1.4.

The roadmap may be written in advance, but future rules should not become active obligations until their trigger conditions are met.

---

## 2. Current Active Version

```text
v1.4 = Self-Management, AI-Assisted Execution, and Client Value Control
```

v1.4 controls:

1. Paul’s daily priority.
2. Idea capture.
3. Issue size classification.
4. ChatGPT / Codex / Antigravity role control.
5. AI execution boundaries.
6. Client value gate.
7. Constitution review trigger.

---

## 3. v1.5 Candidate

### Name

Client Maintenance and Monthly Report Commitment Rule

### Trigger

Write or activate v1.5 when Optimaks needs to explain monthly service value to clients.

### Conditions

1. Monthly fee scope needs to be stated clearly.
2. Monthly health report format becomes stable.
3. Maintenance tasks need boundaries.
4. Bug fix, backup, uptime, content update, data export, and support rules need formal definition.
5. Client cancellation and handover rules need to be clear.

### Example Content

1. What monthly fee includes.
2. What monthly fee excludes.
3. Monthly system health report format.
4. Maintenance scope.
5. SLA or response expectation.
6. Data export and handover rule.
7. Small update vs paid CR boundary.

---

## 4. v2.0 Candidate

### Name

Commercial Delivery Constitution

### Trigger

Write or activate v2.0 when Optimaks is ready to formally pitch, quote, contract, collect payment, and deliver to clients.

### Conditions

1. Starter / Problem / Growth packages are defined.
2. Proposal and quotation templates are ready.
3. SOW, contract, PDPA, and T&C are ready.
4. E-sign workflow is ready.
5. Payment milestones are defined.
6. Client onboarding and handover workflow are active.

---

## 5. v3.0 Candidate

### Name

Multi-Client and Multi-Industry Template Constitution

### Trigger

Write or activate v3.0 when Optimaks operates beyond one industry template.

### Conditions

1. Aircon OS is no longer the only active industry template.
2. Core system and industry templates need strict separation.
3. Multiple clients need standardized deployment.
4. Customization requests need governance.
5. Template library needs version control.
6. Multi-client data isolation rules are required.

---

## 6. v4.0 Candidate

### Name

Team and ISO-Ready Scale Constitution

### Trigger

Write or activate v4.0 when Optimaks starts involving people other than Paul in regular operations.

### Conditions

1. Contractors, engineers, UI designers, sales partners, or admin support are involved.
2. Role and responsibility matrix is required.
3. Approval matrix is required.
4. Code review and access control rules are required.
5. Incident response process is required.
6. ISO-ready evidence control becomes active.

---

## 7. Principle

Future constitution versions may be planned early, but they should become active only when the operating reality requires them.

```text
Roadmap can be written early.
Active constitution should follow real operating maturity.
```
