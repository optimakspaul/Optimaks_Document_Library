# CHK-DEV-008 v1.5 AI and MVP Readiness Checklist

Version: v1.5.1 WAS Positioning Patch  
Related Standards: STD-DEV-020, STD-DEV-021, STD-DEV-022

---

## 1. AI Context Readiness

- [ ] Constitution version identified.
- [ ] Project context provided.
- [ ] Business objective written.
- [ ] In-scope items defined.
- [ ] Out-of-scope items defined.
- [ ] User flow provided.
- [ ] Page map provided.
- [ ] Component map provided.
- [ ] Data model draft provided.
- [ ] Acceptance criteria provided.
- [ ] Testing method provided.
- [ ] Forbidden changes listed.

## 2. MVP Readiness

- [ ] MVP objective is clear.
- [ ] MVP is classified as dependent or independent.
- [ ] Small Core is defined.
- [ ] Excluded features are listed.
- [ ] Next-layer candidates are noted.
- [ ] Dependency map is complete.

## 3. Internal vs Client Delivery Check

- [ ] This is clearly marked as internal MVP or client delivery candidate.
- [ ] If client delivery, QA checklist is required.
- [ ] If client data exists, PDPA checklist is required.
- [ ] If production deployment is needed, deployment checklist is required.
- [ ] If handover is needed, handover checklist is required.
- [ ] If recurring fee exists, monthly value evidence is defined.

## 4. GitHub Control

- [ ] GitHub Issue or CR exists.
- [ ] Branch name is defined.
- [ ] Expected changed files are listed.
- [ ] Diff review is required before merge.
- [ ] Changelog update is required.
