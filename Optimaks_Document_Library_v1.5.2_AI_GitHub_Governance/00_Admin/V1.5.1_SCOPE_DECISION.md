## 1. Why v1.5.1 Exists

After v1.5 Cleanup + Fusion, an additional constitution-level positioning rule was identified:

> Optimaks does not build generic SaaS products. Optimaks builds industry-specific Workflow Automation Systems and SME Operating OS solutions.

This rule is important enough to enter the constitution because it affects:

1. Company positioning.
2. Product scope.
3. MVP planning.
4. Client proposal direction.
5. AI prompt context.
6. Module prioritization.
7. What Optimaks should and should not build.

---

## 2. Added to Constitution

The following ideas were added:

1. Optimaks is not a generic SaaS company.
2. Optimaks builds industry-specific WAS / SME Operating OS solutions.
3. Projects should begin with industry workflow discovery.
4. The goal is to reduce repetitive admin coordination, not claim full human replacement.
5. A project must define industry, workflow, pain point, Small Core MVP, data flow, and value proof before implementation.
6. Market research details should not be mixed into the core constitution.

---

## 3. Not Added to Constitution Core

The following topics are useful, but should remain outside the core constitution unless they become repeated governance rules:

1. Microsoft Dynamics 365 setup cost comparison.
2. JobStreet / SEEK Admin Executive job posting analysis.
3. Admin Executive salary comparison.
4. Competitor pricing and package pricing.
5. One-off sales scripts.
6. Short-term market observations.
7. Detailed proposal copy.

These can be stored in project documents, sales playbooks, market research notes, or proposal templates.
