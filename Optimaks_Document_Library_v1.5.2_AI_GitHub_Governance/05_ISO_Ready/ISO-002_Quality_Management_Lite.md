## Quality Management Lite

Optimaks should keep the process practical while still controlled.

Minimum quality controls:

1. Written scope before development.
2. Change Request before major changes.
3. GitHub Issue for development tasks.
4. Branch for risky changes.
5. Preview / Staging before production.
6. QA checklist before launch.
7. Changelog and version history.
8. Handover record for clients.

## Principle

The system should be light enough to use daily, but strong enough to prove professionalism.
