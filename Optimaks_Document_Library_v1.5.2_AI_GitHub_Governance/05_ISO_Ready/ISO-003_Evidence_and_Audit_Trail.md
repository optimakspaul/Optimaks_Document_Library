## Evidence Sources

1. GitHub Issues
2. Change Request Register
3. Commit history
4. Pull requests / branch diffs
5. Vercel Preview links
6. QA reports
7. Deployment checklist
8. Client acceptance note
9. Handover guide
10. Maintenance report

## Rule

Every significant change should be traceable to a reason, a task, an implementation, a test, and a release.
