# 00_Admin

Administrative records for the Optimaks Document Library v1.4.3 Integrated package.

## Key Files

- `DOC-INDEX-001_Document_Index.md` — master document index for the current library.
- `CHANGELOG.md` — version-by-version change record.
- `VERSION_HISTORY.md` — detailed evolution from v0.1 to v1.4.3 Integrated.
- `VERSION_DELTA_SUMMARY_v1.3_to_v1.4.3.md` — summary of what changed from v1.3 to v1.4.3.
- `README_v1.4.3_UPDATE_INSTRUCTIONS.md` — GitHub upload and update instructions for this package.
- `DOCUMENT_REVIEW_REPORT_v1.4.3.md` — review report covering duplication, merge, deletion, and amendment recommendations.
- `REFERENCE_FULL_SINGLE_FILE_v1.4.3.md` — single-file reference backup for the integrated version.
- `REFERENCE_FULL_SINGLE_FILE_v1.3.md` — historical reference backup for the v1.3 baseline; keep as archive, do not treat as the active version.
- `GITHUB_DESKTOP_CLEANUP_GUIDE.md` — cleanup guide for messy GitHub Desktop uploads.
- `FILE_TREE.txt` / `FILE_TREE_v1.4.3.txt` — expected file tree snapshots.
- `manifest.json` — machine-readable file manifest.

## Rule

The active repository entry point is the root `README.md`. Historical v1.3 files may be kept for traceability, but active development should follow v1.4.3 Integrated unless a future version supersedes it.
