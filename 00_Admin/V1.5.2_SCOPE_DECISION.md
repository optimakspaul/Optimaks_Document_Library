## 1. Reason for Patch

During MVP-01 preparation, Optimaks formalized a repeatable AI-assisted development model using:

```text
ChatGPT
Codex
Antigravity
GitHub Issues
Feature branches
Pull Requests
Vercel Preview
Human QA
```

This is not a one-time project note. It affects repeated development execution, safety, traceability, code review, staging, and client delivery readiness.

Therefore it qualifies as a constitution-level patch under `STD-DEV-014`.

---

## 2. Added to Constitution

The following are added as formal governance rules:

```text
1. One Issue / One Branch / One PR Rule
2. Codex / Antigravity Tool Boundary Rule
3. AI Code Human Review Rule
4. Batch Scope Boundary Rule
5. Client Workspace Data Isolation Rule
6. Preview Before Production Rule
7. Issue / Changelog Traceability Rule
8. Conflict Prevention Rule for multiple AI tools
```

## 4. Not Added as Constitution Core

The following remain project-level execution details unless they become repeated governance needs:

```text
- Exact MVP-01 batch numbering beyond current project
- Individual Codex prompt wording for one issue
- UI-specific design choices for one screen
- Temporary setup commands that may change by framework version
- One-off debugging notes
```

## 5. Version Decision

This is a patch release, not a minor release.

```text
v1.5.1 = Product positioning patch

```

It strengthens execution governance without changing the company product positioning or delivery model.
